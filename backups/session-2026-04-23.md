### Tasks Completed
- Diagnosed Talos 500 loop: LM Studio returns HTTP 500 (not context overflow) when context is large, so hermes was retrying 3x on the same bloated payload and dying instead of compressing
- Confirmed Talos is NOT on Rolling Window (Truncate Middle in LM Studio, per prior config)
- Patched `D:/hermes-agent/agent/error_classifier.py`: 500/502 from local providers (custom, lmstudio, ollama, llamacpp, local) with tokens > 50% of context window now reclassifies as `context_overflow` with `should_compress=True` instead of `server_error`
- Result: retry loop will compress and retry instead of burning all 3 retries and halting

### Decisions
- Reclassify as context_overflow (not just add should_compress flag) - semantically correct, triggers the full compression+step-down path not just a retry
- 50% token threshold is appropriate floor - prevents compressing on genuine transient 500s at low context
- Remote providers untouched - their 500s still get standard server_error treatment
