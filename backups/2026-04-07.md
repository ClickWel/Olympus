## Decisions

- gemma4-26b uses Ollama registry base, not LM Studio GGUF. Update whenever Ollama adds native gemma4 arch support.
- Role switcher confirmation instruction belongs in CLAUDE.md, never in role files.
- Script paths in role files: always quoted forward slashes. Backslashes cause path concatenation against working directory.
- 32k ctx is optimal for 4070 Ti + gemma4-26b. Weights fit in 12GB VRAM, KV cache overflow goes to 64GB system RAM.
- CPU layer offload not needed - model fits fully on GPU at this quant level.

## Key Learnings

- Ollama 0.20.3 does not support gemma4 architecture. Registry pull works; LM Studio GGUF does not.
- Local models process CLAUDE.md on every turn but may hallucinate file reads. Behavioral instructions must be in CLAUDE.md.
- Unquoted backslash paths in role files get concatenated to working directory by local models.
