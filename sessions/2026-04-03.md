## What Happened

### Talos Launcher Debugging
- Original bat was using direct `claude.exe --model qwen3.5:latest` - failed model validation
- Tried: ANTHROPIC_AUTH_TOKEN=ollama, --bare flag, --custom-model-id (unknown option), ANTHROPIC_DEFAULT_SONNET_MODEL override - all failed
- Root cause: Claude Code validates --model against hardcoded Anthropic model list regardless of ANTHROPIC_BASE_URL
- Fix: `ollama launch claude --model <model>` bypasses validation entirely - this is the only working method
- Added wt.exe wrapper via launch-talos.ps1 for styled purple tab

### Key Technical Finding
- `ollama launch claude` sets ANTHROPIC_AUTH_TOKEN=ollama (confirmed via binary string extraction)
- That alone doesn't fix it when a stored /login key exists - auth conflict is cosmetic, not the blocker
- The real bypass mechanism is internal to ollama's launcher - not fully reverse-engineered
- Workaround: bat calls ollama launch claude via ps1 script, not claude.exe directly
