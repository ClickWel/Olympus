## [14:xx] - Microsoft Todo fix landed

**Problem:** `azure-identity`'s `DeviceCodeCredential.get_token()` and MSAL's `acquire_token_by_device_flow` both hang when run inside a subprocess - the device code URL/code never reaches the user and the call blocks indefinitely. Token cache at `~/.cache/msal/` was never written.

**Fix:**
- Separated auth into a standalone `auth` command that blocks intentionally (user runs it once)
- Added `msal.SerializableTokenCache` backed by `~/.cache/msal/todo_cache.json`
- All other commands (`list`, `add`, `done`) load from cache and refresh silently - no blocking
- `todo_manager.py` and `enhanced_todo_manager.py` are superseded; `simple_todo_manager.py` is the one that works

**Auth flow (one-time setup):**
1. Run `python "D:/Clawdbot/skills/microsoft-todo/scripts/simple_todo_manager.py" auth`
2. Visit https://www.microsoft.com/link, enter the printed code
3. Token cached at `C:/Users/click/.cache/msal/todo_cache.json`
4. All future calls are silent

**If it breaks again - check these first:**
- Cache file exists: `C:/Users/click/.cache/msal/todo_cache.json`
- If missing or expired: re-run `auth` command
- Refresh tokens expire after 90 days of inactivity
- Do NOT use `azure-identity`'s `DeviceCodeCredential` - it swallows the prompt. Use `msal.PublicClientApplication` directly.
- `tenant_id="consumers"` is correct for personal Microsoft accounts

**Usage:**
```
python "D:/Clawdbot/skills/microsoft-todo/scripts/simple_todo_manager.py" auth   # one-time
python "D:/Clawdbot/skills/microsoft-todo/scripts/simple_todo_manager.py" list
python "D:/Clawdbot/skills/microsoft-todo/scripts/simple_todo_manager.py" add "task title"
python "D:/Clawdbot/skills/microsoft-todo/scripts/simple_todo_manager.py" done "partial title or number"
```
