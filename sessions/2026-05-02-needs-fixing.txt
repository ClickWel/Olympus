‚óŹ The compression auxiliary block has model: '' and provider: auto. When Hermes
  tries to auto-compress, it needs to call another model to do the
  summarization. If that auxiliary model call fails or stalls, compression
  doesn't happen, the context keeps growing, and eventually LM Studio gets a
  request that's too big and returns 500.

  This is the chain:
  1. Session grows past 60% threshold
  2. Hermes tries to compress using summary_model: openrouter/auto
  3. That OpenRouter call either fails, is slow, or the compression logic isn't
  triggering properly mid-agent-run
  4. Context keeps growing past 100% of loaded_context_length
  5. LM Studio returns 500
  6. Hermes then runs /compress reactively after the failure

  The fix: set an explicit, reliable summary model and lower the threshold so
  compression fires earlier and more aggressively. What's your OpenRouter status
   right now - is it working reliably? And do you want compression to stay on
  OpenRouter or switch to something local?
