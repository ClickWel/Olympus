## Root Causes Identified

### 1. reasoning_content sent to LM Studio (PRIMARY)
**File:** `/home/click/.hermes/hermes-agent/run_agent.py` ~line 8718

Hermes stores reasoning as `msg["reasoning"]`. When building the API payload it copied this to `reasoning_content` for ALL providers including local/custom. GLM5.1's jinja template processes `reasoning_content` on every assistant message by injecting `<think>` blocks. In a 418-message session this corrupts the entire prompt and LM Studio returns 500.

**Fix:** Only set `reasoning_content` when provider is NOT local. Strip it for `custom`, `lmstudio`, `ollama`, `llamacpp`, `local`.

### 2. 500 classified as server_error instead of context_overflow (SAFETY NET)
**File:** `/home/click/.hermes/hermes-agent/agent/error_classifier.py` ~line 506

HTTP 500 from LM Studio was classified as `server_error` - burns 3 retries and dies. For local providers it should trigger compress-and-retry.

**Fix:** When `status_code in (500, 502)` and provider is local, return `context_overflow` with `should_compress=True`.
