‚óŹ Good research. Here's the clean picture:

  The key finding: there's no way to inject a prompt into an already-running
  OpenCode terminal session from outside. Kimaki always spawns a new session -
  that's why it interferes with active CLI work.

  But there's a better option: opencode serve + HTTP.

  OpenCode has a built-in HTTP server mode. You run opencode serve in the
  background, and any external trigger (Discord ping, script, whatever) POSTs a
  prompt to it via HTTP. It processes it and streams back the response. That's
  how Remote-OpenCode works.

  The clean plan for you:

  Each agent runs two things:
  1. Their normal OpenCode terminal (what you use now - your interactive
  session)

   to Discord-triggered tasks

  When a Discord ping arrives, a small Python watcher script catches it and
  POSTs to that agent's serve port. The agent handles it in the background
  without touching your active terminal session. Zero interference.
